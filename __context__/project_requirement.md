# 软件需求说明书

## 1. 项目概述
### 1.1 项目背景
本项目旨在创建一个介绍弗洛伦丝·南丁格尔生平的静态网站。南丁格尔（1820年5月12日-1910年8月13日）是英国护士和统计学家，被誉为现代护理学的创始人。本网站的目标是向普通大众介绍南丁格尔的生平事迹、贡献和影响，提高公众对这位历史人物的认识和了解。

## 2. 用户需求
### 2.1 用户角色
* 普通用户：对南丁格尔生平感兴趣的一般读者，可能包括学生、成年人和老年人。

### 2.2 用户故事
1. 作为一个普通用户，我想要通过时间线的方式了解南丁格尔的生平，以便能够清晰地把握她人生的重要事件。
2. 作为一个普通用户，我希望能够浏览与南丁格尔相关的历史图片，以便更直观地了解她的生平和那个时代的背景。
3. 作为一个普通用户，我想要阅读南丁格尔的名言，以便更好地理解她的思想和理念。
4. 作为一个普通用户，我希望了解南丁格尔对现代护理的影响，以便理解她的历史意义。

## 3. 项目功能需求
### 3.1 项目功能概述
本静态网站主要包含以下四个核心功能模块：
1. 南丁格尔生平时间线
2. 图片库
3. 南丁格尔名言集
4. 南丁格尔对现代护理的影响

### 3.2 功能需求详述
1. 南丁格尔生平时间线
   - 功能描述：以图文并茂的方式展示南丁格尔人生的重要事件
   - 实现方式：
     1. 使用HTML和CSS创建静态时间线
     2. 所有事件信息预先编写到HTML中
   - 内容要求：
     - 时间线应覆盖南丁格尔的整个生平（1820-1910）
     - 每个事件的描述应简洁明了，不超过100字
     - 包含至少10个关键事件，例如：
       1. 1820年5月12日：出生于意大利佛罗伦萨
       2. 1844年：决定成为护士，遭到家人反对
       3. 1851年：在德国凯撒斯韦特接受护理培训
       4. 1853年：成为伦敦慈善医院的护士长
       5. 1854年10月21日：与38位护士志愿者前往克里米亚战场
       6. 1855年：改善斯库台医院的卫生条件，大幅降低死亡率
       7. 1859年：出版《护理札记》
       8. 1860年：在伦敦圣托马斯医院成立护士学校
       9. 1907年：获得英国国王爱德华七世颁发的功绩勋章
       10. 1910年8月13日：在伦敦逝世

2. 图片库
   - 功能描述：展示南丁格尔的照片、手稿和相关历史图片
   - 实现方式：
     1. 使用HTML和CSS创建图片网格布局
     2. 所有图片和说明文字预先嵌入HTML中
   - 内容要求：
     - 包含至少15张与南丁格尔相关的历史图片，例如：
       1. 南丁格尔的肖像
       2. 克里米亚战争期间的照片
       3. 南丁格尔绘制的东部军队死亡原因统计图
       4. 爱伯利公园（Embley Park）的照片
       5. 南丁格尔的坟墓照片
       6. 南丁格尔在医院工作的场景重现
       7. 南丁格尔的手稿或著作封面
     - 每张图片都应有简要说明文字
     - 图片分辨率应适合网页浏览，建议最大宽度不超过1200像素

3. 南丁格尔名言集
   - 功能描述：展示南丁格尔的著名言论
   - 实现方式：
     1. 使用HTML和CSS创建名言列表或卡片布局
     2. 所有名言预先编写到HTML中
   - 内容要求：
     - 包含至少20条南丁格尔的名言，可以从她的著作《护理札记》、《医院札记》等中摘录
     - 每条名言应标注出处或背景信息

4. 南丁格尔对现代护理的影响
   - 功能描述：简要介绍南丁格尔的贡献及其现代意义
   - 实现方式：
     1. 使用HTML创建文章页面
     2. 内容直接编写在HTML中，包括文字和嵌入的图片
   - 内容要求：
     - 文章应包含但不限于以下几个方面：
       1. 护理教育：建立世界上第一个非修道院形式的护士学校
       2. 医院管理：改善医院卫生条件，降低死亡率
       3. 公共卫生：推动公共卫生改革，促成1875年英国公共卫生法
       4. 护理伦理：提升护士的社会地位和职业形象
       5. 统计学贡献：发展极坐标形式的圆饼图（南丁格尔玫瑰图）
       6. 国际影响：指导美国护理先驱琳达·理查德兹，影响全球护理发展
     - 文章长度建议在2000-3000字之间
     - 包含至少3张相关图片，用于补充说明文字内容

### 3.3 非功能需求
1. 性能需求
   - 页面加载时间应在3秒内完成（考虑到图片较多）
   - 网站应兼容主流浏览器（Chrome, Firefox, Safari, Edge）

2. 设计需求
   - 网站设计应简洁明了，易于导航
   - 使用响应式设计，确保在桌面和移动设备上都能良好显示

3. 内容需求
   - 所有文字内容应直接写入HTML文件中
   - 图片应存储在网站目录中，使用相对路径引用

4. 技术要求
   - 仅使用HTML、CSS和少量JavaScript（如果必要）
   - 不使用服务器端语言或数据库
   - 所有页面和资源文件应可以直接通过文件系统或简单的Web服务器访问
